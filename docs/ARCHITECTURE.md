# 链动小铺比价台架构规划

## 目标

输入链动小铺店铺链接后，系统自动读取店铺信息、商品名称、价格、库存、原始分类和商品链接，并在统一页面里做搜索、筛选、排序、分组比价和跳转。

## 模块

1. 前端工作台
   - `/` 是只读比价页面。
   - `/admin` 是管理后台，需要输入管理密码。
   - 店铺录入、删除、单店刷新、全部刷新。
   - 定时刷新开关与刷新间隔配置。
   - 界面语言在管理后台配置，前端文案集中在 `src/i18n.js`。
   - 管理后台展示当前 Git 版本、远端版本和更新状态，并提供自更新入口。
   - 分组概览卡片：展示每个关注分类的商品数、有库存数量和最低有货价。
   - 商品比价表：按关键词、店铺、类型、库存、系统分组筛选，按价格、库存、更新时间、分组排序。
   - 商品分组可在表格中手动修改，修改后持久保存。
   - 商品可在管理后台手动删除。删除只是本地隐藏，手动刷新店铺或手动刷新全部会重新拉回；定时自动刷新不会恢复隐藏商品。
   - 点击店铺或商品时跳转到链动小铺原始页面。

2. 后端 API
   - 校验并规范化 `https://pay.ldxp.cn/shop/{token}`。
   - 调用链动小铺公开接口：
     - `/shopApi/Shop/info`
     - `/shopApi/Shop/categoryList`
     - `/shopApi/Shop/goodsList`
   - 上游返回 HTML 反爬验证页时，后端识别 `arg1` 并计算 `acw_sc__v2` cookie 后重试。
   - 标准化外部字段，隐藏前端对第三方接口变化的依赖。
   - 根据原始分类、商品标题和描述进行关键词分组。
   - 支持 `店铺 token + 商品 key` 维度的手动分组覆盖，刷新商品后继续生效。
   - 支持 `店铺 token + 商品 key` 维度的本地删除隐藏；自动刷新保留隐藏记录，管理后台手动刷新会恢复。
   - 写操作要求 `x-admin-password` 请求头；管理密码优先读取环境变量 `ADMIN_PASSWORD`，未设置时读取 `data/shops.json` 的 `settings.adminPassword`，默认值为 `admin123`。管理后台可修改本地配置密码；设置接口不会回传密码。
   - 版本检查和自更新接口同样要求 `x-admin-password`，自更新会拒绝有未提交本地改动的部署。
   - 将店铺、商品、分类、刷新配置持久化到本地 JSON 文件。
   - 使用 UTF-8 读写本地 JSON 数据，支持通过 `LDXP_DATA_DIR` 指定数据目录。

3. 分组规则
   - 当前关注分组：`GPT Free`、`GPT Team`、`GPT Plus`、`GPT Pro`、`Claude`、`Grok`、`Gemini / Google`、`邮箱`、`接码`、`其他`。
   - 原始分类和商品标题是强信号，描述是弱信号。
   - 当商品同时命中多个词时，按命中得分和优先级归入最明确的分组。例如标题含“GPT Free 已接码”的商品归入 `GPT Free`，而标题只描述接码服务的商品归入“接码”。Kiro 相关商品归入 `Claude`。
   - API、中转、接口类商品默认排除；邮箱 API 取件、OAuth、Graph 令牌等邮箱工具类商品保留在“邮箱”组。

4. 定时刷新
   - 默认开启，间隔 15 分钟。
   - 后端每 30 秒检查一次是否到达刷新窗口。
   - 每次刷新会覆盖当前快照，不记录历史曲线。

5. 数据存储
   - 当前使用 `data/shops.json`。
   - `productGroupOverrides` 保存手动商品分组覆盖。
   - `deletedProducts` 保存本地隐藏商品。
   - 后续如果要做多人管理、账号权限、价格历史，可以迁移到 SQLite。

## 数据流

```mermaid
flowchart LR
  A["管理页输入店铺链接"] --> B["POST /api/shops"]
  B --> C["解析店铺 token"]
  C --> D["读取 shop info"]
  D --> E["读取 categoryList"]
  E --> F["分页读取 goodsList"]
  F --> G["标准化字段与关键词分组"]
  G --> H["写入 data/shops.json"]
  H --> I["GET /api/shops"]
  I --> J["前端分组比价展示"]
```
